# 目标：Java → Go 后端逐步迁移

## 目标陈述

将现有 Java 后端（djclub-backend）的 15 个业务域逐步迁移到 Go 后端（Cgame/backend），保持 API 100% 兼容，支持数据无缝迁移，最终完全替换 Java 后端。迁移按域分批进行，优先独立域，最后迁移核心域，预计周期 4-6 个月。

## 共享理解

详见 [facts.md](facts.md)，包含 10 条已确认的事实：
- 迁移顺序：独立域先行，核心域最后
- 数据库策略：共用 PostgreSQL，Go 新增表，Java 旧表逐步弃用
- API 兼容性：100% 兼容，前端零改动
- 双写策略：Java 主写 → Go 主写 → Java 下线
- 验证方式：数据校验脚本 + 接口对比测试 + 灰度放量
- 单体部署，暂不需要微服务基础设施

## 执行计划

详见 [plan.md](plan.md)，包含 8 个 Phase 的详细步骤：

1. **Phase 1**（第 1-2 周）：访客分析、文件存储、系统配置
2. **Phase 2**（第 3-4 周）：通知消息、聊天客服
3. **Phase 3**（第 5-8 周）：用户中心、选手生态
4. **Phase 4**（第 9-12 周）：商品中心、营销引擎
5. **Phase 5**（第 13-18 周）：订单核心、支付收银
6. **Phase 6**（第 19-22 周）：财务结算
7. **Phase 7**（第 23-26 周）：游戏互动、外部集成
8. **Phase 8**（第 27-28 周）：系统权限、Java 下线

每个域按 6 步标准流程执行：DDL & Migration → Model & Repository → Service & DTO → Handler & Routing → 双写 & 数据同步 → 切流 & 下线。

## 完成条件

- 所有 15 个业务域的 API 路由已在 Go 端实现并通过测试
- 每个域的数据校验脚本持续通过（行数/checksum 一致）
- 接口对比测试通过率 100%
- Java 服务完全停止，Go 成为唯一后端
- 前端管理端、小程序、官网均无需修改即可正常工作
