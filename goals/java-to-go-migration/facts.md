# Facts

- 迁移按域逐步进行，优先选择独立性强的域（访客分析、文件存储、系统配置）作为首批迁移对象
- 迁移期间 Go 与 Java 共用同一个 PostgreSQL 数据库实例，Go 端新增表，Java 端旧表逐步弃用
- Go 端 API 保持 100% 兼容 Java 端，包括路由路径、请求/响应格式、错误码，前端和客户端无需修改
- 双写策略：Java 主写，Go 异步同步数据；数据校验通过后切 Go 主写，Java 降级为只读；最后 Java 完全下线
- 每个域迁移完成后必须通过数据校验脚本（行数/checksum）、接口对比测试、灰度放量三层验证
- 整体迁移周期预计 4-6 个月，15 个业务域，每个域耗时 1-2 周
- 保持单体部署模式，暂不需要服务发现、API 网关、服务网格等微服务基础设施
- 迁移范围仅限后端 API，前端管理端、移动端 App、小程序、官网均不在此次迁移范围内
- Go 端模块间禁止直接 import model/table，必须通过接口解耦或领域事件跨模块通信
- Go 端数据库表结构需与 Java 端兼容，支持软删除（deleted 字段）、乐观锁（version 字段）、统一时间戳命名
